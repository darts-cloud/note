## 案出し

### Context
モバイルSuicaの利用履歴を週1回自動取得し、月ごとの使用状況のまとめと改善策を提案する仕組みを構築したい。
私は「管理作業は高コスト」と考えており、極力「頭を使わず、手だけ動かす」状態を目指している。

### Goals
1. Playwright（Python）を使用したモバイルSuica会員サイトのスクレイピングスクリプトを作成する。
2. ログインは「セッション保持（ユーザーデータディレクトリ使用）」方式を採用し、2回目以降のログイン入力を省略する。
3. 取得したデータを `suica_history.json` または `csv` として保存し、Claude Codeがいつでも解析できる状態にする。
4. 保存されたデータを読み込み、「月別の支出まとめ」と「無駄遣いへの対策」を出力する解析プロンプト/スクリプトを準備する。

### Requirements
- OS: Ubuntu 24.04 LTS
- Language: Python
- Library: Playwright (ヘッドレス/有頭の切り替えが容易なため)
- Strategy:
    - 最初に1回だけ有頭ブラウザでログインし、認証状態を保存する手順を含めること。
    - サイト構造の変化に備え、セレクタは極力シンプルに保つ。
    - 取得項目：利用日、利用内容（駅名/種別）、支払金額、残高。

### Task Details
1. まず、Playwrightのセットアップと、ログインセッションを保存するための初期設定スクリプトを作成してください。
2. 次に、履歴画面からデータを抽出するメインスクリプトを作成してください。
3. 最後に、蓄積されたデータを「面（長期的・多角的）」で分析し、改善案を提示するテンプレートを作成してください。

### Output
まずは、環境構築に必要なコマンドの実行と、セッション保存用のスクリプトの作成から着手してください。
