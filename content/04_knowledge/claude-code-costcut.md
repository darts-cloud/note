# AI開発のコストを最適化する：Claude Code/Cursor時代の賢い「脳メモリ」節約術

AIツール（Claude Code, Cursor等）は非常に強力ですが、無計画に使い続けるとAPIコスト（トークン消費）が膨らみます。

本ドキュメントでは、支出を抑えつつ、開発効率を最大化するための具体的な対策をまとめました。

---

## 1. コンテキスト（文脈）のダイエット
AIは「読み込んだ情報の量」に応じて課金されます。不要な情報を読み込ませないことが最大の節約です。

- **`.cursorrules` や `.claudignore` の活用**: 
  ビルド成果物（dist, build）、ログ、依存ライブラリ（node_modules）などは必ず除外設定を行い、AIが「見なくていいもの」を明確にします。

- **ファイルを選択的に開く**: 
  プロジェクト全体を読み込ませるのではなく、修正に必要な最小限のファイルだけをコンテキストに含めます。「今からこの3つのファイルを修正する」と宣言するだけで、無駄な検索（RAG）を抑制できます。

## 2. 「手だけ動かす」と「AIに任せる」の境界線
管理作業（計画、申請、依頼）は「高コスト作業」です。これをAIで効率化しつつ、APIコストを抑えるバランスが重要です。

- **テンプレートの徹底活用**: 
  よく使うプロンプトや、リポジトリの構造説明はテンプレート化しておきます。ゼロから指示を考えるコストと、試行錯誤によるトークン消費の両方を削減できます。

## 3. 「点」ではなく「面」で指示を出す
一つの修正（点）を繰り返すと、その度にコンテキストが蓄積され、後半になるほど1発あたりのコストが上がります。

- **バッチ処理的な依頼**: 
  「この変数を変えて」「次にここを変えて」と細切れに頼むのではなく、「今回の目的は〇〇なので、関連するA, B, Cのファイルを一括で修正して」と、全体像（面）を見せて一度に処理させます。

- **要約とリセット**: 
  やり取りが長くなったら、一度「これまでの修正内容を要約して」と頼み、その要約を元に新しいチャットセッションを開始します。蓄積された過去の不要なログを切り捨てることで、トークン消費をリセットできます。

```
/new-chat: 現在の会話を終了し、新しい会話を開始します。
/clear: ターミナルの表示をクリアします（会話履歴のリセットとは異なります）。
/compact: 会話を要約してコンテキストを圧縮し、トークンを節約します。
```

## 4. 開発環境の最適化
ツールそのものの設定で、無駄な通信を抑えます。

- **インデックス作成の制御**: 
  大規模なプロジェクトでは、フルインデックス作成を避け、必要なディレクトリのみを対象にします。

- **モデルの使い分け**: 
  単純なリファクタリングやテストコードの生成には、軽量で安価なモデル（Claude 3 Haiku等）を使い、複雑なロジック設計にのみ最強モデル（Claude 3.5 Sonnet等）を投入する使い分けを意識しましょう。

---

### まとめ：低コスト・高効率なワークフロー
1.  **環境を整える**: `.claudignore` で不要な情報を遮断。
2.  **まず手作業**: 難しいと感じたら、無理にAIを使わず10秒だけ手を動かす。
3.  **面で捉える**: 目的を明確にし、一括で指示を出す。
4.  **定期的なリセット**: 会話が長くなったら要約して新セッションへ。

これらの習慣と「情報の絞り込み」を組み合わせることで、開発効率を維持しながら、AIコストを最適化し、快適なAI開発を継続できます。
