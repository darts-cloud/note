# idea ディレクトリのルール

## ディレクトリ構造

```
content/idea/
├── idea.md              # アイデアの一覧・メモ置き場
├── 1_DO/                # 実行中のアイデア
├── 2_CHECK/             # 検討・レビュー中のアイデア
├── 3_COMPLATE/          # 完了したアイデア
└── <トピック名>/        # アイデアごとのサブディレクトリ
    ├── 1.案出し.md
    ├── 2.迷っている点.md
    ├── 3.AIとの深掘り.md
    ├── 4.課題.md
    └── 5.決定内容.md
```

## アイデアの追加ルール

1. アイデアのトピックごとにサブディレクトリを作成する（例：`ETロボコン/`）
2. サブディレクトリ内に以下の5ファイルを番号順に作成する
3. ファイルは番号順に埋めていく（1→2→3→4→5）

## 各ファイルの役割

| ファイル | 役割 | 書くべき内容 |
|---|---|---|
| `1.案出し.md` | ブレインストーミング | 制約なしにアイデアや背景を箇条書き |
| `2.迷っている点.md` | 不確定事項の洗い出し | 決めきれていない点・判断が必要な点 |
| `3.AIとの深掘り.md` | AIとの対話ログ | AIに分析・提案させた結果をそのまま貼る |
| `4.課題.md` | 課題の整理 | 解決すべき課題をカテゴリ別に整理 |
| `5.決定内容.md` | 意思決定の記録 | 最終的に決めたことだけを簡潔に書く |

## フロントマターのルール

全ファイルに以下を付与する。

```yaml
---
title: <番号.ファイル名>
tags:
  - idea
  - <トピック固有のタグ>
---
```

## ステータス管理

- アイデアの進捗は `1_DO` → `2_CHECK` → `3_COMPLATE` の順に移動する
- 各ステータスディレクトリ内の `README.md` にそのステータスのトピック一覧を記載する

## 作業フロー

```
idea.md にメモ
  ↓
トピックディレクトリを作成
  ↓
1.案出し → 2.迷っている点 → 3.AIとの深掘り → 4.課題 → 5.決定内容
  ↓
ステータスを 1_DO → 2_CHECK → 3_COMPLATE に更新
```

## ワークフローの記載
- ワークフローを記載する場合は、視覚的にもわかりやすくなるようにMermaidでも記載すること。
- Mermaidの記載は概要として、詳細を書きすぎないこと。ただ、注意点は書いて欲しい。
