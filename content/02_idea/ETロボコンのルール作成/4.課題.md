## チーム運営

- [ ] 「何をやればいいかわからない」状態を解消するタスクバックログがない
- [ ] マイルストーンの完了定義（Done基準）が曖昧で、進捗判断ができない
- [ ] 購買・備品管理が暗黙知のまま属人化している
- [ ] ベテランと2年目の間でナレッジ移転の仕組みがない

## 開発・Git運用

- [ ] Gitの運用ルール（ブランチ・PR・コミット粒度）が未整備
- [ ] AIツール（Cursor・Google AI Pro）の使用権限・範囲が不明確
- [ ] ライセンス数が限られており、誰がいつ使えるかのルールがない
- [ ] AIが生成したコードのレビュー・品質担保フローがない

## スケジュール

- [x] WBS・マイルストーンは例年ベースで作成済み
- [ ] 各タスクの粒度が大きく、2年目メンバーが着手できない単位になっている可能性

## 方針決定・意思決定

- [x] 「方針が効いているか」を判断する軸（基準・数値目標）が定まっていない
- [x] チームの状況（詰まり・体感）を定期的に吸い上げる仕組みがない
- [x] マイルストーン到達前に兆候を見る早期シグナルの指標が設定されていない
- [x] 効果が出るまで時間がかかるため、正しいか分からないまま次の決断を迫られる → 「仮説として決める」習慣で対処
	- 期限＋仮説で対処

## NotebookLM運用

- [x] アカウント運用方式が未決定 → 全員無料版・管理者が編集者招待
- [x] AI採点結果の公式/参考の扱いが未定 → 参考扱いに決定
- [x] インプット（ソース・PDF）の更新タイミング → 週1・固定担当者が更新
- [x] 採点基準の更新単位 → 過去の高得点モデルをベースに運用中
- [ ] NotebookLM更新の固定担当者が未選定
