# アクティビティ図構成仕様

ETロボコン2026 アプライドクラスの核心である **「走行体と無線通信デバイスの2システム連携」** を記述するため、主要ユースケース `UC-200`（`ETラリーを攻略する`）の内部フローであるサブ機能 `UC-201`（`[オプション] ゲート位置を特定する`）および `UC-202`（`[オプション] ゲート通過経路を計画する`）におけるアクティビティ図の構成仕様（要素と流れ）と、全体の処理フローを表すスイムレーン付きMermaid図を定義します。本アクティビティ図は、主アクターである **「競技者」** の指示を受け、システム内部（走行体と無線通信デバイス）がどのように協調してアクションを進行させるかを詳細に示します。

---

## 1. 2システム協調アクティビティのフロー（Mermaid表現）

競技者、システム境界内の走行体、および無線通信デバイスの3つのパーティション（スイムレーン）における、連携と処理の並行・分岐を示します。

```mermaid
sequenceDiagram
    autonumber
    actor Contestant as 競技者
    participant Robot as 走行体 (HackSPi)
    participant Device as 無線通信デバイス (PC)

    Note over Contestant, Device: 【キャリブレーション・スタート前】
    Contestant->>Device: 1. 復号キー(4桁テキスト)を手入力
    Device->>Device: 2. 復号キーをメモリに記憶保持

    Note over Contestant, Device: 【競技スタート・ETラリー開始】
    Contestant->>Robot: 3. スタート指示 (スタートスイッチ押下)
    Robot->>Robot: 4. ヒントカード1 (赤ゲート) のカメラ認識
    alt 認識成功
        Robot->>Robot: 5. 平文座標を特定・記憶
        Robot->>Robot: 6. ヒントカード2 (青/黄ゲート) のカメラ認識
        alt 認識成功
            Robot->>Robot: 7. Base64暗号文字列を抽出
            Note over Robot, Device: 【2システム情報連携 (メインルート)】
            Robot->>Device: 8. Bluetoothで暗号文字列を送信
            alt 送信成功
                Device->>Device: 9. Base64デコード
                Device->>Device: 10. 登録済みのキーでAES-128復号
                alt 復号成功 (フォーマット正常)
                    Device->>Device: 11. 青・黄ゲートの絶対座標を特定
                    Device->>Robot: 12. Bluetoothで特定座標データを返送
                    Robot->>Device: 13. 経路計画をリクエスト
                    Device->>Device: 14. 衝突回避経路を計算
                    Device->>Robot: 15. 走行用経路コマンドデータを送信
                    Robot->>Robot: 16. 走行コマンドキューを設定し走行開始 (事前計画走行)
                else 復号失敗 (キーミスマッチ等)
                    Device->>Robot: 11a. 復号エラー応答を返送
                    Robot->>Robot: 11b. 自律的カラー追従走行モードに移行 (フォールバック)
                end
            else 送信失敗 (再接続タイムアウト)
                Robot->>Robot: 8b. 自律的カラー追従走行モードに移行 (フォールバック)
            end
        else 認識失敗 (リトライオーバー)
            Robot->>Robot: 6b. 自律的カラー追従走行モードに移行 (フォールバック)
        end
    else 認識失敗 (リトライオーバー)
        Robot->>Robot: 4b. 自律的カラー追従走行モードに移行 (フォールバック)
    end

    Note over Robot: 【ETラリー周回走行 (最大3周回)】
    loop 走行・補正制御
        alt 事前計画走行モード時
            Robot->>Robot: 17. ゲート位置補助情報をカメラ検出・デコード
            Robot->>Robot: 18. 補助情報の絶対座標で自己位置を補正
            Robot->>Robot: 19. 計画経路に従いオドメトリで赤->青->黄ゲートを通過
        else 自律的カラー追従走行モード時
            Robot->>Robot: 17a. カメラ画像から色 (赤・青・黄の順) を検出・重心抽出
            Robot->>Robot: 18a. カラー誘導制御により、リアルタイムでゲートを進入・通過
        end
    end
```

## 2. アクティビティ図構成仕様（UML記述仕様）

### 2.1. パーティション（Swimlane）の定義
* **Swimlane 1**: `競技者 (Contestant)` ── 外部アクター。キャリブレーション時の復号キー入力およびスタート操作、走行監視を行う主体。
* **Swimlane 2**: `走行体 (Robot)` ── システム構成要素（内部）。コース走行とオンボードのセンサ・画像処理を担当。
* **Swimlane 3**: `無線通信デバイス (Device)` ── システム構成要素（内部）。固定設置され、Bluetooth受信データのAES復号および経路計算を支援。
