# アクティビティ図 PlantUML ソースコード

ETラリーにおける **「走行体と無線通信デバイスの2システム連携」** と **「自律的カラー追従走行への二重化フォールバック設計」** を表す、スイムレーン（競技者、走行体、無線通信デバイス）付きUMLアクティビティ図の PlantUML ソースコードです。

## 1. PlantUML コード

```plantuml
@startuml
skinparam monochrome true
skinparam shadowing false
skinparam packageStyle rect
skinparam defaultTextAlignment center

' スイムレーン（パーティション）の定義
|競技者\n(Contestant)|
|走行体 (Robot)\n(HackSPi)|
|無線通信デバイス\n(PC)|

' 開始
|競技者\n(Contestant)|
start
:復号キー(4桁テキスト)を手入力\n(キャリブレーション時);

|無線通信デバイス\n(PC)|
:復号キーをメモリに記憶保持;

|競技者\n(Contestant)|
:スタート指示\n(スタートスイッチ押下);

|走行体 (Robot)\n(HackSPi)|
:走行を開始し\nETラリーへ進入;
:「ヒントカード1」を検出・キャプチャ;

if (画像認識・デコード成功?) then (yes)
    :平文テキストを解析し\n赤色ゲート座標を保存;
    :「ヒントカード2」を検出・キャプチャ;
    
    if (画像認識・デコード成功?) then (yes)
        :二次元コードから\nBase64の暗号文字列を取得;
        :Bluetoothで暗号文字列を送信;
        
        |無線通信デバイス\n(PC)|
        if (規定時間(1秒)内に受信成功?) then (yes)
            :受信したBase64文字列をデコード;
            :安全なメモリ領域で\nAES-128 (ECB) 復号を実行;
            
            if (復号結果のフォーマット正常?) then (yes)
                :青・黄ゲートの絶対座標を確定;
                :Bluetooth通信経由で\n座標データを走行体に返送;
                
                |走行体 (Robot)\n(HackSPi)|
                :座標データを受信し\n全3ゲートの座標を記憶;
                :無線通信デバイスへ\n経路計画をリクエスト;
                
                |無線通信デバイス\n(PC)|
                :衝突回避を考慮した\n最短ゲート通過経路を計算;
                :生成した経路コマンド列を\n走行体に送信;
                
                |走行体 (Robot)\n(HackSPi)|
                :経路コマンドを\n走行コマンドキューにロード;
                :計画経路に従い\nオドメトリ制御で走行開始;
                
                repeat
                    :ゲート位置補助情報(二次元コード)を\nカメラで検出・読み取り;
                    :絶対座標を用いて\n自己位置(オドメトリ)を補正;
                    :計画ルートに従い\n赤→青→黄ゲートを順番に通過走行;
                repeat while (規定周回(3周回)を完了したか?) is (no)
                ->yes;
                
                :次のエリア(またはガレージ)へ\n向け移動し攻略完了;
                stop
                
            else (no - 復号失敗/キーミスマッチ)
                :走行体に復号エラー応答を返送;
                |走行体 (Robot)\n(HackSPi)|
                goto fallback
            endif
        else (no - 通信タイムアウト)
            |走行体 (Robot)\n(HackSPi)|
            goto fallback
        endif
    else (no - ヒント2認識失敗)
        |走行体 (Robot)\n(HackSPi)|
        goto fallback
    endif
else (no - ヒント1認識失敗)
    |走行体 (Robot)\n(HackSPi)|
    label fallback
    :自律的カラー追従走行モードを有効化\n(二重化フォールバック);
    
    repeat
        :搭載カメラを用いてターゲット色\n(赤→青→黄の順)を探索・重心抽出;
        :リアルタイムカラー誘導制御を用いて\n検出したゲートを自律的に進入・通過;
    repeat while (規定周回(3周回)を完了したか?) is (no)
    ->yes;
    
    :次のエリア(またはガレージ)へ\n向け移動し攻略完了;
    stop
endif

@enduml
```

## 2. アクティビティ図の主要設計ポイント

1. **3つのスイムレーンによる「2システム連携」の可視化**  
   主アクターである「競技者（アクター）」、システム境界内の「走行体 (Robot)」、および「無線通信デバイス (PC)」の3つに明確に境界（パーティション）を分け、オンボード処理（画像認識・オドメトリ制御）と固定設置PC処理（AES復号・経路計算）の間のBluetooth通信によるデータのやり取りの分担を厳密に表現しています。

2. **自律的カラー追従走行への二重化フォールバック (`goto fallback`)**  
   - 「ヒント1認識失敗」
   - 「ヒント2認識失敗」
   - 「通信タイムアウト (Bluetooth接続エラー)」
   - 「復号失敗 (キーミスマッチ)」
   
   といった主要な故障モード (FM-101〜103) のすべての分岐において、即座に安全かつ頑健な **`自律的カラー追従走行モード`** へ遷移するフォールバック制御設計を論理的に表現しています。

3. **競技時間軸との厳密な整合**  
   ユースケース名と同様に、スタートスイッチ押下（スタート指示）の後に、走行体が動き出してヒントカード検出を行う時間的順序が視覚的にも厳密に守られています。
