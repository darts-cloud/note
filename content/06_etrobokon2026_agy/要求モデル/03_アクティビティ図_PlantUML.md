# アクティビティ図 PlantUML ソースコード

ETラリーにおける **「走行体と無線通信デバイスの2システム連携」** と **「自律的カラー追従走行への二重化フォールバック設計」** を表す、スイムレーン付きUMLアクティビティ図の PlantUML ソースコードです。

## 1. PlantUML コード

```plantuml
@startuml
skinparam monochrome true
skinparam shadowing false
skinparam packageStyle rect
skinparam defaultTextAlignment center

' スイムレーン（パーティション）の定義
|競技者\n(Contestant)|
|走行体 (Robot)\n(HackSPi)|
|無線通信デバイス\n(PC)|

' 開始
|競技者\n(Contestant)|
start
:復号キー(4桁テキスト)を手入力\n(キャリブレーション時);

|無線通信デバイス\n(PC)|
:入力された復号キーを記憶保持;

|走行体 (Robot)\n(HackSPi)|
:ETラリーへ進入し\nヒントカード1・2を順番に読み取る;

if (画像認識・デコード成功?) then (yes)
    :読み取った情報を\n無線通信デバイスへ送信;
    
    |無線通信デバイス\n(PC)|
    if (Bluetooth通信成功 (1.0秒以内)?) then (yes)
        :復号キーを用いて青・黄ゲート座標を特定し\n最適通過経路を計算;
        
        if (復号成功 (フォーマット正常)?) then (yes)
            :最適経路コマンドデータを走行体に返送;
            
            |走行体 (Robot)\n(HackSPi)|
            :デバイスから送信された最適経路をロード;
            
            repeat
                :コース上の「ゲート位置補助情報」を順次読み取り\n自己位置を自動補正;
                :最適経路に従いオドメトリ制御で\n赤→青→黄の順にゲートを物理通過;
            repeat while (規定周回(3周回)を完了したか?) is (no)
            ->yes;
            
            :ガレージまたは次のエリアへ向け走行し\nETラリー攻略を完了;
            stop
            
        else (no - 復号エラー応答)
            :走行体に復号エラー応答を送信;
            |走行体 (Robot)\n(HackSPi)|
            goto fallback
        endif
    else (no - 通信タイムアウト)
        |走行体 (Robot)\n(HackSPi)|
        :自動再接続を試行 (3.0秒以内);
        if (再接続成功?) then (yes)
            ' 再接続成功時は送信処理を再試行
        else (no)
            goto fallback
        endif
    endif
else (no - 画像認識失敗)
    |走行体 (Robot)\n(HackSPi)|
    label fallback
    :自律的カラー追従走行モードを有効化\n(二重化フォールバック);
    
    repeat
        :カメラ画像からターゲット色を探索し\nリアルタイムカラー誘導でゲートを進入・通過;
    repeat while (規定周回(3周回)を完了したか?) is (no)
    ->yes;
    
    :ガレージまたは次のエリアへ向け走行し\nETラリー攻略を完了;
    stop
endif

@enduml
```

## 2. アクティビティ図の主要設計ポイント

1. **3つのスイムレーンによる「2システム連携」の可視化**  
   主アクターである「競技者（アクター）」、システム境界内の「走行体 (Robot)」、および「無線通信デバイス (PC)」の3つに明確に境界（パーティション）を分け、オンボード処理（画像認識・自己位置補正走行）と固定設置PC処理（復号・経路計算）の間のBluetooth通信によるデータのやり取りの分担を厳密に表現しています。

2. **走行体の物理動作を主軸としたスリムな制御フロー**  
   メモリ内部の細かな変数退避や、AES-128暗号といった「裏方処理」の表記を徹底して排除し、走行体自身が「物理的にヒントカードを検出する」「通信する」「自己位置補正しながらゲートを通過する」といった物理・協調動作にフォーカスを当てることで、モデルの可読性と一貫性を高めています。

3. **自律的カラー追従走行への二重化フォールバック (`goto fallback`)**  
   - 「ヒントカード画像認識失敗」
   - 「通信タイムアウト (Bluetooth接続エラー・3.0秒再接続失敗)」
   - 「復号失敗 (キーミスマッチ)」
   
   といった主要な故障モード (FM-101〜103) のすべての分岐において、即座に安全かつ頑健な **`自律的カラー追従走行モード`** へ遷移するフォールバック制御設計を論理的に表現しています。
