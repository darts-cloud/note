# Appendix: 03_UMLシーケンス図指示ガイド

本ファイルは、UML/SysMLシーケンス図（Sequence Diagram：MermaidまたはPlantUML）をAIに作成・修正・レビューさせる際に読み込ませる「指示ガイド」です。

---

## 1. コピペ用AIプロンプト

シーケンス図の設計をAIに指示する際は、以下の枠内のテキストをそのままコピーして送信してください。

```markdown
シーケンス図（MermaidコードまたはPlantUMLコードを含む）を作成・修正するにあたり、表記の厳密性と機能実現の妥当性を高めるため、以下の【シーケンス図設計厳守ルール】を徹底してください。

### 【シーケンス図設計厳守ルール】

1. 同期・非同期メッセージの厳密な使い分け
   - 走行体（Robot）と無線通信デバイス（Device）の協調、または異なるスレッド／タスク間でメッセージを送信する場合、同期通信（呼び出し側が応答を待つ）と非同期通信（送り先の処理を待たずに進める）を正しく区別して表現すること。
   - 【表記ルール】:
     - 同期メッセージ: 閉じられた塗りつぶし矢印（`->` / `->`）を使用する。
     - 非同期メッセージ: 開いたハーフ矢印（`->>` / `->>`）を使用する。通常、別タスクの起動や通信電文の送信に用いる。

2. 非同期メッセージ送信後の応答（返り値）ラインの省略
   - **非同期メッセージ（開いた矢印）を送信した直後に、点線矢印（返り値）を戻してはなりません。** 非同期メッセージは「投げっぱなし」の通信であるため、応答メッセージは文法上不要です。
   - もし送信先の処理結果を後で受け取る必要がある場合は、非同期に応答メッセージを別メッセージ（コールバック等）として送信先から送信元へ新しく矢印を伸ばす形で表現すること。

3. 複合フラグメント（Alt / Opt / Loop）のガード条件の明示
   - 条件分岐（alt）やオプション実行（opt）、繰り返し（loop）が発生する箇所には、必ず境界条件やイベント名をブラケット `[条件式]` で明示すること。ガード条件のない分岐は設計として不完全であり、減点対象となります。

4. 相互作用の再利用（ref）による階層化・可読性の最大化
   - 1つのシーケンス図の中にすべての詳細な処理を描き込もうとすると、図面が横・縦に肥大化し、A3での可読性が著しく低下します。
   - 意味のある処理のまとまりは、積極的に `ref`（相互作用の再利用）フレームを用いて抽象化・別図面化し、全体シーケンスのシンプルさを保つこと。
```
